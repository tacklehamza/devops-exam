5. Docker Optimization — Best Practices Applied [2 Marks]

  • Best Practice 1 — Slim Base Image: Used node:18-alpine instead of node:18. This reduces image size from ~900MB to ~180MB.
  • Best Practice 2 — Layer Caching: Copied package*.json first before copying source code. This means npm install only re-runs when dependencies change, not on every code change.
  • Best Practice 3 — .dockerignore: Created .dockerignore to exclude node_modules, .env, and .git from the build context, reducing build time and image size.
  • Best Practice 4 — Production Dependencies Only: Used --omit=dev flag to skip dev dependencies in the final image.

deployment.yaml:

PART 4: Troubleshooting and DevOps Analysis [4 Marks]

Issue: Git Push Rejected Due to Remote Conflict

1. Issue Identified:

When trying to push local commits to GitHub, the push was rejected because the remote repository had commits that were not present locally.

2. Error Message (Before Fix):

! [rejected]        main -> main (fetch first)
error: failed to push some refs to 'https://github.com/tacklehamza/devops-exam.git'
hint: Updates were rejected because the remote contains work that you do not
hint: have locally. Use 'git pull' before pushing again.
[ Screenshot: Terminal showing git push rejected error ]

3. Diagnosis:

The remote repository had a README.md and initial commit added via GitHub web interface. The local repository had different commits. Git rejected the push to prevent overwriting remote history.

4. Fix Applied:

git pull origin main --allow-unrelated-histories
git add .
git commit -m "Merge remote changes"
git push origin main

5. Result After Fix:

Push was successful. All local and remote commits were merged and pushed to GitHub.

[ Screenshot: Terminal showing successful git push after fix ]
